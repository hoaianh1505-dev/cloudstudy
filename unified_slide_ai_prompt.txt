### Slide 2: Đặt vấn đề - Giới hạn của lưu trữ máy chủ cục bộ (Disk Storage Limits)
* **Tiêu đề**: Thách Thức Trong Việc Lưu Trữ Tài Liệu Trên Máy Chủ Cục Bộ
* **Nội dung hiển thị**:
  * **Bộ nhớ hữu hạn**: Ổ cứng SSD của máy chủ chạy ứng dụng sẽ bị quá tải nhanh chóng khi sinh viên tải lên lượng lớn tài liệu PDF và video.
  * **Rủi ro mất dữ liệu**: Máy chủ ảo trên Render sử dụng Ephemeral Storage (ổ đĩa tạm thời) - dữ liệu tự động bị xóa sạch mỗi lần reboot/deploy.
  * **Nghẽn băng thông server**: Khi nhiều người tải file lớn đồng thời, CPU và đường truyền của server chính sẽ bị nghẽn làm gián đoạn hệ thống.
* **Ý tưởng thiết kế**: Sơ đồ một máy chủ cục bộ bị đầy bộ nhớ (vạch đỏ cảnh báo 99% Full) so sánh với biểu tượng đám mây lưu trữ rộng mở.

### Slide 3: Tổng quan giải pháp - Kiến trúc tích hợp AWS S3 & MongoDB
* **Tiêu đề**: Giải Pháp Kiến Trúc Tách Biệt Máy Chủ & Lưu Trữ Đám Mây
* **Nội dung hiển thị**:
  * **Máy chủ ứng dụng (Stateless)**: NodeJS Server chỉ xử lý logic nghiệp vụ và định tuyến yêu cầu, không lưu file vật lý.
  * **Hạ tầng lưu trữ (AWS S3)**: Đảm nhận 100% nhiệm vụ lưu giữ và truyền tải tệp tĩnh gốc nhị phân.
  * **Cơ sở dữ liệu (MongoDB)**: Chỉ lưu trữ thông tin liên kết (metadata) siêu nhẹ từ S3.
* **Ý tưởng thiết kế**: Sơ đồ 3 thành phần chính: Web Server NodeJS <---> MongoDB Atlas (lưu metadata) <---> AWS S3 (lưu file thô). Đặt file ảnh "multi_cloud_architecture_visual.png" làm hình minh họa chính cho slide này.

  * **Lưu trữ & Truyền dữ liệu**: Tải lên an toàn, Stream Proxy tải xuống, đọc PDF trực tuyến.
  * **Tương tác thông minh**: Tìm kiếm tài liệu và thảo luận cùng Trợ lý ảo AI.

### Slide 5: Tính năng 1 - Hệ thống Xác thực & Cách ly dữ liệu người dùng
* **Tiêu đề**: Xác Thực Phiên Làm Việc & Bảo Mật Dữ Liệu Cá Nhân
* **Nội dung hiển thị**:
  * **Mã hóa mật khẩu**: Sử dụng thuật toán băm `bcryptjs` để bảo mật mật khẩu lưu trong cơ sở dữ liệu.
  * **Signed Cookies**: Server gửi về trình duyệt một Cookie chứa ID Session đã được ký mật mã chống giả mạo bằng khóa bí mật.
  * **Cách ly tài nguyên**: Bộ lọc Middleware đảm bảo sinh viên chỉ truy cập được tài liệu của riêng mình.
* **Ý tưởng thiết kế**: Đồ họa mô tả luồng xác thực: Đăng nhập -> Sinh Session -> Ký Cookie -> Trả về Client -> Xác thực thành công truy cập tài nguyên.

### Slide 6: Tính năng 2 - Dashboard tổng quan & Thống kê lưu trữ thời gian thực
* **Tiêu đề**: Giao Diện Dashboard Thống Kê Lưu Trữ Cloud
* **Nội dung hiển thị**:
  * **Đếm thời gian thực**: Tự động đếm tổng số thư mục và số lượng file tài liệu đang lưu trên S3.
  * **Biểu thị phần trăm (Capacity Graph)**: Biểu đồ thanh tiến trình hiển thị tỷ lệ dung lượng đã sử dụng trên tổng định mức Cloud cấp phát.
  * **Quy đổi tự động (Dynamic Bytes)**: Sử dụng thuật toán tự động quy đổi kích thước file thô từ Bytes sang các đơn vị thân thiện (KB, MB, GB).
* **Ý tưởng thiết kế**: Ảnh chụp màn hình Dashboard EJS thực tế với hiệu ứng Glassmorphism kính mờ sang trọng, nổi bật là biểu đồ dung lượng lưu trữ đám mây.

### Slide 7: Tính năng 3 - Không gian quản lý tài liệu theo thư mục phân cấp
* **Tiêu đề**: Tổ Chức Không Gian Thư Mục Học Tập Ngăn Nắp
* **Nội dung hiển thị**:
  * **Không gian phân cấp**: Hỗ trợ tạo cấu trúc thư mục học tập lồng nhau (Năm học > Học kỳ > Môn học).
  * **Thanh Breadcrumb điều hướng**: Tự động hiển thị đường dẫn thư mục hiện tại để sinh viên dễ dàng chuyển đổi nhanh.
  * ** AJAX Rename**: Hỗ trợ đổi tên thư mục nhanh chóng ngay trên giao diện web bằng AJAX.
* **Ý tưởng thiết kế**: Ảnh chụp giao diện quản lý thư mục môn học thực tế, làm nổi bật cấu trúc liên kết và thanh Breadcrumb dẫn đường.

### Slide 8: Tính năng 4 - Quy trình Tải tài liệu lên AWS S3 (Secure Upload)
* **Tiêu đề**: Luồng Tải Lên Tài Liệu Không Lưu File Tạm
* **Nội dung hiển thị**:
  * **Memory Storage (Multer)**: NodeJS Server chặn file bằng Middleware Multer, lưu tệp tạm thời dưới dạng Buffer trong RAM.
  * **Đẩy trực tiếp lên S3**: Server lập tức gọi AWS SDK v3 chuyển dữ liệu Buffer này lên S3.
  * **Lợi ích**: Không ghi file tạm xuống đĩa cứng server, giảm thiểu Disk I/O, tăng tốc độ xử lý upload tệp.
* **Ý tưởng thiết kế**: Sơ đồ luồng upload: Browser -> [Multer (RAM Buffer)] -> [NodeJS Server] -> (AWS SDK) -> AWS S3. Không có bước ghi đĩa cứng cục bộ (Local Disk).

### Slide 9: Demo giao diện tải tài liệu lên (Upload Document UI Demo)
* **Tiêu đề**: Demo Trực Quan Giao Diện Tải Tài Liệu Lên Cloud
* **Nội dung hiển thị**:
  * **Khu vực kéo thả (Drag and Drop)**: Hỗ trợ người dùng kéo thả file hoặc chọn file từ máy tính dễ dàng.
  * **Thanh tiến trình trực quan (Upload Progress)**: Hiển thị tiến trình đẩy file lên cloud theo thời gian thực (Real-time).
  * **Thông báo tức thì**: Phản hồi thành công ngay khi S3 xác nhận đã nhận file, tự động làm mới danh sách tài liệu.
* **Ý tưởng thiết kế**: Ảnh chụp giao diện UI trang tải file lên với khu vực kéo thả viền đứt nét nổi bật và hiệu ứng thanh tiến trình màu xanh ngọc.

### Slide 10: Bộ lọc kiểm soát định dạng & Giới hạn dung lượng tệp
* **Tiêu đề**: Bộ Lọc Kiểm Soát Tài Nguyên File Tải Lên Đám Mây
* **Nội dung hiển thị**:
  * **Giới hạn dung lượng**: Cấu hình bộ lọc giới hạn kích thước tối đa 10MB cho mỗi tệp tải lên để tiết kiệm tài nguyên S3.
  * **Bộ lọc định dạng (File Filter)**: Cho phép các định dạng học tập an toàn:
    * Tài liệu & Văn bản: PDF, DOCX, PPTX, XLSX, ZIP, TXT, MD.
    * Hình ảnh & Video: JPG, JPEG, PNG, WEBP, MP4, MKV, AVI, MOV.
  * **Bảo mật**: Chặn hoàn toàn file thực thi nguy hiểm (như `.exe`, `.bat`, `.js`) để bảo vệ hạ tầng server.
* **Ý tưởng thiết kế**: Sơ đồ khối lọc file đầu vào: Client gửi file -> Kiểm tra dung lượng (<10MB?) -> Kiểm tra định dạng -> Hợp lệ -> Đẩy lên S3.

### Slide 11: Thiết lập bảo mật AWS S3 Private & Phân quyền IAM Policy
* **Tiêu đề**: Thiết Lập Bảo Mật S3 Bucket Private & AWS IAM
* **Nội dung hiển thị**:
  * **Block Public Access**: Kích hoạt tính năng khóa toàn diện mọi quyền truy cập công khai từ bên ngoài Internet vào bucket `cloudstudy-2026`.
  * **IAM User chuyên dụng**: Tạo một User con riêng biệt `cloudstudy-dev-new` chỉ phục vụ việc chạy mã nguồn.
  * **Đặc quyền tối thiểu (Least Privilege)**: IAM User chỉ có đúng 3 quyền cơ bản: `s3:PutObject` (Tải lên), `s3:GetObject` (Tải xuống), `s3:DeleteObject` (Xóa) trên Bucket chỉ định.
* **Ý tưởng thiết kế**: Sơ đồ một chiếc khóa bảo mật có nhãn "IAM Policy", chỉ mở được đúng chiếc hộp S3 Bucket `cloudstudy-2026` chỉ định, các hộp tài nguyên khác bị khóa chặt.

### Slide 12: Cơ chế xác thực lập trình tới S3 Bucket qua HMAC SigV4
* **Tiêu đề**: Cơ Chế Khởi Tạo & Xác Thực Kết Nối AWS S3
* **Nội dung hiển thị**:
  * **Cấu hình an toàn (.env)**: Sử dụng các biến `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_BUCKET_NAME` và `AWS_REGION` (`ap-southeast-2` - Sydney).
  * **Ký số Signature Version 4 (SigV4)**: Server không gửi Secret Key trực tiếp. Thay vào đó, SDK dùng Secret Key để ký số HMAC mã hóa các yêu cầu HTTP gửi sang AWS.
* **Ý tưởng thiết kế**: Sơ đồ luồng: NodeJS Server (sử dụng Access Key ID & Secret) -> Sinh mã ký số HMAC -> AWS S3 giải mã xác minh -> Phản hồi dữ liệu tệp tin.

### Slide 13: Thiết kế cơ sở dữ liệu lưu trữ Metadata (MongoDB Document Schema)
* **Tiêu đề**: Lược Đồ Dữ Liệu Document Schema Tham Chiếu S3
* **Nội dung hiển thị**:
  * **Trường `s3Key` & `s3Url` (String)**: Lưu trữ khóa định danh S3 Key và đường dẫn liên kết URL đầy đủ tới file.
  * **Trường `fileName` (String)**: Tên gốc của tệp tin hiển thị cho sinh viên.
  * **Trường `fileSize` & `fileType`**: Kích thước file tính bằng Bytes và định dạng MIME của tệp tin.
  * **Trường `folderId` (ObjectId)**: Liên kết tới Folder cha chứa tài liệu này.
* **Ý tưởng thiết kế**: Hình ảnh cấu trúc một Document Schema trong Mongoose ODM, làm nổi bật kiểu dữ liệu của các trường `s3Key`, `fileName`, `fileSize` và các khóa liên kết MongoDB.

### Slide 14: Tính năng 5 - Tải xuống tài liệu bảo mật qua Stream Proxy
* **Tiêu đề**: Kiến Trúc Download Tệp An Toàn Bằng NodeJS Proxy
* **Nội dung hiển thị**:
  * **Không để lộ S3 URL**: Trình duyệt người dùng chỉ thấy liên kết trung gian của Web Server (Ví dụ: `/documents/download/:id`).
  * **Đọc dữ liệu dạng Luồng (Stream)**: Server Node.js gọi `GetObjectCommand` lấy luồng dữ liệu (`ReadableStream`) từ S3 về.
  * **Pipe Stream**: Sử dụng hàm `stream.pipe(res)` truyền trực tiếp dữ liệu thô về trình duyệt, giải phóng RAM server tối đa.
* **Ý tưởng thiết kế**: Đặt file ảnh sơ đồ luồng tải xuống "s3_architecture_diagram.png" (hoặc "s3_visual_architecture_1782834328077.png") làm hình minh họa chính cho slide này.

### Slide 15: Tính năng 6 - Xem trước PDF & Hình ảnh trực tuyến (Viewer UI Demo)
* **Tiêu đề**: Trình Xem Tài Liệu Trực Tuyến Tải Từ AWS S3
* **Nội dung hiển thị**:
  * **Embedded PDF Viewer**: Nhúng trực tiếp file PDF trong iframe giao diện web, hỗ trợ đọc bài giảng trực tiếp không cần tải tệp về thiết bị.
  * **Hiển thị hình ảnh trực tiếp (Image Preview)**: Tự động render định dạng ảnh JPG, PNG, WEBP từ S3 proxy ngay trên trình duyệt.
  * **Bảo mật**: Không hiển thị đường dẫn gốc của AWS S3 trong thuộc tính nguồn (`src`) của thẻ iframe hay thẻ image.
* **Ý tưởng thiết kế**: Ảnh chụp màn hình trang chi tiết tài liệu học tập PDF hiển thị mượt mà trên trình duyệt web, không có dấu vết của đường dẫn S3 công khai.

### Slide 16: Tính năng 7 & 8 - Tìm kiếm nhanh toàn văn & Xóa tài liệu đồng bộ
* **Tiêu đề**: Tìm Kiếm Thông Minh & Quy Trình Xóa Đồng Bộ
* **Nội dung hiển thị**:
  * **Tìm kiếmRegex**: Sử dụng biểu thức chính quy (Regex) của MongoDB quét nhanh qua database để tìm kiếm không dấu/không biệt chữ hoa-thường.
  * **Quy trình xóa đồng bộ**: Lấy `s3Key` từ MongoDB -> Gọi dịch vụ AWS S3 xóa file thô bằng `DeleteObjectCommand` -> Xóa vĩnh viễn bản ghi metadata khỏi MongoDB Atlas.
  * **Tính toàn vẹn dữ liệu**: Nếu xóa file trên S3 lỗi, hệ thống chặn không cho xóa metadata trong DB để tránh hiện tượng "tệp mồ côi".
* **Ý tưởng thiết kế**: Sơ đồ luồng: Click Xóa file -> Lấy s3Key -> Gọi API S3 xóa file thô -> Xác nhận thành công -> Gọi MongoDB xóa metadata -> Phản hồi kết quả.

### Slide 17: Tính năng 9 - Trợ lý học tập AI hỗ trợ phân tích tài liệu trực tiếp

  * **Các tiện ích nhanh**: Hỗ trợ tóm tắt ý chính của bài giảng, giải thích công thức phức tạp, dịch thuật tiếng Anh chuyên ngành.
  * **Giao diện Split Screen**: Sinh viên vừa đọc PDF bên trái, vừa chat hỏi đáp với AI bên phải vô cùng trực quan.
* **Ý tưởng thiết kế**: Đặt file ảnh "ai_study_assistant_ui_mockup.png" làm hình minh họa chính cho slide này.

### Slide 18: Đề xuất Tính năng Đột phá & Ưu việt của Cloud Study
* **Tiêu đề**: Định Hướng Phát Triển Tính Năng Đám Mây Đột Phá
* **Nội dung hiển thị**:
  * **Tự động hóa Metadata bằng AI**: Tự động trích xuất nội dung bài giảng, tóm tắt ý chính và sinh nhãn gán thẻ (tags) ngay khi lưu file thô lên S3.
  * **Chia sẻ an toàn tự hủy (Pre-signed URL)**: Sinh đường dẫn liên kết tải tạm thời tự hủy sau 1 giờ hoặc giới hạn số lần tải để chia sẻ dữ liệu tuyệt đối an toàn.
  * **Quản lý phiên bản & So sánh (AI Version Diff)**: Sử dụng S3 Versioning lưu vết lịch sử tệp tin và dùng AI để liệt kê các điểm sửa đổi giữa các phiên bản.
  * **Tự động tối ưu chi phí (Glacier Lifecycle)**: Tự động phân tích tần suất sử dụng tệp để chuyển tài liệu cũ xuống Archive Glacier giúp tiết kiệm 90% chi phí.
* **Ý tưởng thiết kế**: Sơ đồ trung tâm thể hiện 4 tính năng đột phá: AI Metadata, Pre-signed URL, AI Version Diff và S3 Glacier, kết nối với logo của AWS S3 và MongoDB Atlas.
