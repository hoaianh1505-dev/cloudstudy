Kiến Trúc Sử Dụng Dịch Vụ AWS

Tài liệu kỹ thuật giải thích cách tích hợp, phân quyền và điều hướng dữ liệu của các dịch vụ đám mây AWS dùng trong dự án CloudStudy.

💡

Tổng quan kiến trúc

Dự án sử dụng dịch vụ Amazon Simple Storage Service (AWS S3) làm hệ thống lưu trữ đối tượng đám mây (Cloud Object Storage) chính để quản lý tất cả các tài liệu bài giảng, tệp tin và video học tập. Toàn bộ quyền truy cập S3 được quản lý chặt chẽ thông qua AWS IAM (Identity and Access Management) để đảm bảo tệp tin được bảo mật và chỉ truy cập được bởi những người dùng hợp lệ.
📐

💡 Giải thích thuật ngữ cho giảng viên:
  • AWS S3 (Simple Storage Service): Nơi lưu trữ vật lý các file tài liệu. Bucket được thiết lập ở chế độ **Private hoàn toàn** (Block all public access).
  • AWS IAM (Identity and Access Management): Quản lý danh tính và phân quyền. Một **IAM Policy** được định nghĩa để giới hạn quyền tối thiểu cho server (chỉ được Ghi, Đọc, và Xóa đối tượng; không có quyền xóa Bucket). Một **IAM User** đại diện cho NodeJS Server được cấp **Access Key ID** và **Secret Access Key** để ký xác thực kết nối.

Cơ Chế Bảo Mật & Piping Stream

Mô hình bảo mật của CloudStudy sử dụng kiến trúc Reverse Proxy để phân phối file an toàn:

  • Không để lộ URL S3 trực tiếp: Địa chỉ URL của S3 Bucket hoàn toàn được giấu kín. Người dùng bên ngoài chỉ nhìn thấy địa chỉ API của máy chủ Express (ví dụ: /document/12345/download).
  • Kiểm soát quyền truy cập tại Server: Máy chủ NodeJS Express sẽ nhận request, kiểm tra phiên đăng nhập (Session) của sinh viên trong cơ sở dữ liệu. Nếu người dùng hợp lệ, Server mới gửi lệnh lấy tệp từ AWS S3.
  • Tối ưu hóa tài nguyên (Piping Stream): Khi tệp tin được trả về từ S3 dưới dạng luồng dữ liệu (ReadableStream), server không ghi file ra đĩa cứng server mà sử dụng hàm stream.pipe(res) để truyền trực tiếp dữ liệu theo thời gian thực về trình duyệt của khách hàng. Cơ chế này giúp server tiêu thụ bộ nhớ RAM tối thiểu (chỉ vài Kilobytes) kể cả khi truyền tải tệp lớn hoặc stream video dài.

Luồng Tải Lên (Upload File)

Quy trình tải tệp tin của sinh viên lên S3 thông qua các thành phần công nghệ:

1
Client gửi tệp tin
Trình duyệt client đóng gói tệp tin trong request gửi lên API POST /upload.
2
Multer đệm dữ liệu vào RAM
Thư viện multer (Memory Storage) đệm tệp dưới dạng Buffer trong bộ nhớ tạm thời của máy chủ.
3
Gửi dữ liệu lên AWS S3
s3Service tạo lệnh PutObjectCommand chứa file Buffer, định nghĩa ContentType và gửi trực tiếp lên S3 bucket thông qua AWS SDK v3.
4
Lưu Metadata vào MongoDB
Sau khi S3 trả về mã thành công, server tiến hành lưu trữ Metadata tệp (tên tệp, s3Key, kích thước, định dạng, owner) vào database MongoDB để quản lý hiển thị.

Luồng Preview (Xem Trước) & Xem tài liệu

Ứng dụng tích hợp trình xem trước tài liệu trực tiếp trên trang chi tiết nhằm nâng cao trải nghiệm học tập:

  • Hình ảnh: Render trực tiếp bằng thẻ <img src="/document/:id/download">.
  • Tài liệu PDF: Nhúng bằng thẻ <iframe src="/document/:id/download#toolbar=0"> giúp đọc tài liệu dễ dàng, ẩn thanh công cụ nguyên bản để hạn chế tải lén tài liệu.
  • Video học tập: Sử dụng thẻ HTML5 <video controls src="/document/:id/download"> cho phép tua và stream bài giảng trực tuyến.
🤖 Trợ lý AI đọc tài liệu:
Khi người dùng kích hoạt "Cho phép AI đọc tệp", API `/ai/chat` sẽ lấy luồng stream của tệp từ S3.
  • Đối với file text/code: Đọc dạng chuỗi văn bản và chèn vào prompt của Gemini.
  • Đối với file PDF/Hình ảnh: Chuyển đổi sang chuỗi Base64 và truyền dưới dạng dữ liệu đa phương tiện (`inlineData`) để Gemini tự động đọc hiểu trực tiếp.
