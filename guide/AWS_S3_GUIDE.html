Thiết Lập AWS S3

Tài liệu hướng dẫn trực quan từng bước bằng tiếng Việt, giúp bạn dễ dàng cấu hình và kết nối S3 Bucket cho dự án CloudStudy.

1

Tạo mới một S3 Bucket

Nơi lưu trữ tất cả các tài liệu, bài giảng, hình ảnh và video cho hệ thống CloudStudy.

  • Đăng nhập vào trang quản trị AWS: AWS Management Console
  • Nhập S3 vào ô tìm kiếm ở thanh trên cùng và chọn dịch vụ S3.
  • Click vào nút Create bucket (màu cam).
  • Điền các thông tin cơ bản:
    • Bucket name: Nhập một tên duy nhất toàn cầu (Ví dụ: cloudstudy-2026). Chú ý không chứa ký tự viết hoa hoặc khoảng trắng.
    • AWS Region: Chọn vùng máy chủ gần Việt Nam nhất để tối ưu tốc độ truyền tải file. Khuyên dùng: ap-southeast-1 (Singapore) hoặc ap-southeast-2 (Sydney).
    • Object Ownership: Giữ nguyên mặc định là ACLs disabled (Khuyên dùng).
  • Tại mục Block Public Access settings for this bucket:
    • TÍCH CHỌN ô "Block all public access".
    • Giải thích: Để bảo mật tài liệu nội bộ của học sinh, chúng ta chặn hoàn toàn quyền truy cập trực tiếp từ công chúng. Server NodeJS sẽ làm trung gian xác thực đăng nhập rồi mới cấp phát link tạm thời để hiển thị.
  • Cuộn xuống dưới cùng và nhấn nút Create bucket để hoàn thành.
2

Tạo chính sách bảo mật IAM (IAM Policy)

Giới hạn quyền hạn của Server. NodeJS chỉ được phép Đọc, Ghi, Xóa file trong Bucket được chỉ định, tuyệt đối không có quyền quản trị cấp cao để phá hủy Bucket.

  • Nhập IAM vào ô tìm kiếm AWS Console và click dịch vụ IAM.
  • Ở danh sách menu bên trái, click vào Policies.
  • Click vào nút Create policy.
  • Chuyển sang tab JSON (phía bên phải trình soạn thảo) và thay thế toàn bộ nội dung bằng đoạn code dưới đây:
IAM Policy JSON 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "CloudStudyS3Access",
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject"
      ],
      "Resource": [
        "arn:aws:s3:::cloudstudy-2026",
        "arn:aws:s3:::cloudstudy-2026/*"
      ]
    }
  ]
}
QUAN TRỌNG:
Hãy nhớ thay thế cụm từ cloudstudy-2026 trong đoạn mã trên bằng chính xác tên Bucket mà bạn đã tạo ở Bước 1.
  • Click nút Next: Tags, rồi click Next: Review.
  • Đặt tên cho chính sách này dễ quản lý (Ví dụ: CloudStudyS3Policy).
  • Click nút Create policy để lưu lại chính sách.
3

Tạo tài khoản kết nối (IAM User) & Lấy Access Key

Tạo thông tin định danh đặc biệt để dự án Node.js có thể ký tên truy xuất dữ liệu một cách an toàn từ máy chủ khác.

  • Vẫn ở trong dịch vụ quản lý IAM, click chọn mục Users ở menu bên trái.
  • Click nút Add users (Thêm người dùng).
  • Điền tên người dùng: cloudstudy-server.
  • KHÔNG TÍCH CHỌN quyền đăng nhập vào AWS Management Console. Click Next.
  • Tại trang Set permissions (Gán quyền):
    • Chọn mục Attach policies directly (Gán trực tiếp chính sách).
    • Nhập tên Policy bạn vừa tạo ở Bước 2 (CloudStudyS3Policy) vào ô tìm kiếm.
    • Tích chọn vào ô vuông bên cạnh chính sách đó. Click Next.
  • Review thông tin và click nút Create user ở dưới cùng.
  • Khi giao diện trở lại danh sách User, click vào tên User cloudstudy-server bạn vừa tạo.
  • Chuyển qua tab Security credentials.
  • Cuộn xuống mục Access keys và click vào nút Create access key.
  • Tại giao diện chọn nhu cầu, tích chọn Application running outside AWS. Click Next.
  • Đặt một nhãn tên mô tả (Ví dụ: server-connection-key) rồi nhấn Create access key.
LƯU Ý KHÓA BẢO MẬT (CỰC KỲ QUAN TRỌNG):
Màn hình sẽ hiển thị Access Key ID và Secret Access Key.
  • Nhấn nút Download .csv file để tải file chứa các mã này về máy tính.
  • Sao chép và lưu trữ chuỗi bảo mật Secret Access Key ra một file nháp tạm thời. Đây là lần duy nhất AWS cho phép bạn xem thông tin bảo mật này.
4

Cấu hình luật chia sẻ nguồn gốc (CORS)

Quy tắc CORS giúp cho trình duyệt web của học sinh có thể tải file PDF và stream video từ máy chủ S3 một cách mượt mà mà không bị chặn lỗi Cross-Origin.

  • Quay lại giao diện dịch vụ S3 và click vào tên Bucket của bạn.
  • Chuyển qua tab Permissions (Quyền hạn) ở hàng tab ngang bên dưới tên Bucket.
  • Cuộn xuống dưới cùng đến mục Cross-origin resource sharing (CORS) và chọn Edit.
  • Copy chính xác đoạn cấu hình chuẩn JSON dưới đây và dán vào:
CORS Configuration JSON 
[
  {
    "AllowedHeaders": [
      "*"
    ],
    "AllowedMethods": [
      "GET",
      "PUT",
      "POST",
      "DELETE",
      "HEAD"
    ],
    "AllowedOrigins": [
      "*"
    ],
    "ExposeHeaders": [
      "ETag",
      "Content-Type",
      "Content-Length",
      "Content-Disposition"
    ],
    "MaxAgeSeconds": 3000
  }
]
  • Nhấn Save changes để lưu lại luật CORS.
5

Lưu trữ các Khóa & Thiết lập cấu hình ứng dụng

Sau khi hoàn thành 4 bước trên AWS, bạn sẽ có 4 thông số bảo mật quan trọng. Bạn cần lưu trữ các thông số này vào các vị trí dưới đây để ứng dụng hoạt động:

Tên biến (.env) Nguồn lấy thông số Mẫu giá trị thực tế
AWS_REGION Vùng đã chọn ở Bước 1 khi tạo Bucket. ap-southeast-1
AWS_BUCKET_NAME Tên duy nhất của S3 Bucket bạn đã tạo ở Bước 1. cloudstudy-2026
AWS_ACCESS_KEY_ID Mã ID khóa truy cập tải từ file .csv ở Bước 3. AKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEY Mã khóa bí mật sao chép hoặc tải từ file .csv ở Bước 3. wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

1. Cấu hình chạy cục bộ (Local PC)

Tìm tệp tin .env tại thư mục gốc dự án (nếu chưa có, copy từ file .env.example và đổi tên thành .env). Cập nhật 4 dòng cấu hình S3 tương ứng:

AWS_REGION = ap-southeast-1 AWS_ACCESS_KEY_ID = AKIAIOSFODNN7EXAMPLE AWS_SECRET_ACCESS_KEY = wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY AWS_BUCKET_NAME = cloudstudy-2026

2. Cấu hình chạy trên Cloud (Render / Production)

Khi triển khai lên Render, bạn không tải file .env lên Git. Thay vào đó, hãy làm như sau:

  • Truy cập trang quản trị dự án trên dashboard của Render.
  • Chọn dịch vụ Web Service cloud-study của bạn.
  • Nhấp vào mục Environment ở menu bên trái.
  • Nhấn Add Environment Variable và thêm lần lượt 4 cặp khóa - giá trị như bảng ở trên.
  • Nhấn Save changes để Render tiến hành deploy lại với cấu hình mới.
🚨 CẢNH BÁO BẢO MẬT:
Tuyệt đối không đẩy tệp .env lên GitHub để tránh bị lộ mã AWS S3 Access Key. AWS có bot quét tự động, nếu lộ key bạn có thể bị khóa tài khoản hoặc phát sinh chi phí phát ngoài ý muốn.
