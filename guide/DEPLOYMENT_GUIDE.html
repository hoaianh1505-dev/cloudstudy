Hướng Dẫn Triển Khai Lên Render

Tài liệu hướng dẫn từng bước thiết lập tài khoản, đưa mã nguồn lên GitHub và cấu hình chạy thực tế ứng dụng CloudStudy trên đám mây Render.

☁️

Tại sao lại chọn Render?

Render là một nền tảng Cloud Application Hosting hiện đại, hỗ trợ tự động xây dựng và triển khai (CI/CD) ứng dụng Web Service trực tiếp từ các commit trên kho lưu trữ GitHub. Render cung cấp gói dịch vụ miễn phí (Free Tier) phù hợp cho các dự án đồ án học tập và thử nghiệm.
⚙️

Cấu hình Tự động bằng render.yaml

Dự án sử dụng cơ chế Blueprint của Render thông qua tệp cấu hình render.yaml đặt tại thư mục gốc. Khi liên kết Blueprint, Render sẽ tự động nhận diện và thiết lập các tham số môi trường mà không cần nhập tay từng dòng.

Tệp render.yaml gốc 
services:
  - type: web
    name: cloud-study-document-manager
    runtime: node
    buildCommand: npm install
    startCommand: node app.js
    envVars:
      - key: PORT
        value: 3000
      - key: SESSION_SECRET
        sync: false
      - key: MONGODB_URI
        sync: false
      - key: AWS_REGION
        sync: false
      - key: AWS_ACCESS_KEY_ID
        sync: false
      - key: AWS_SECRET_ACCESS_KEY
        sync: false
      - key: AWS_BUCKET_NAME
        sync: false
      - key: GEMINI_API_KEY
        sync: false
🔑

Danh Sách Biến Môi Trường (Environment Variables)

Khi triển khai lên Render, bạn cần cấu hình đầy đủ các biến bảo mật bên dưới để server có thể kết nối với Cơ sở dữ liệu, AWS S3 và Google Gemini:

  • PORT: Mặc định là 3000.
  • SESSION_SECRET: Khóa bảo mật ngẫu nhiên dùng để ký và mã hóa cookie phiên đăng nhập của người dùng. (Nên đặt một chuỗi ký tự dài phức tạp).
  • MONGODB_URI: Chuỗi kết nối kết nối đến cơ sở dữ liệu MongoDB Atlas (ví dụ: mongodb+srv://username:password@cluster...).
  • AWS_REGION: Vùng lưu trữ của S3 bucket (ví dụ: ap-southeast-1).
  • AWS_ACCESS_KEY_ID: ID khóa truy cập của tài khoản AWS IAM User có quyền đọc/ghi trên bucket S3.
  • AWS_SECRET_ACCESS_KEY: Khóa bảo mật tương ứng của IAM User.
  • AWS_BUCKET_NAME: Tên Bucket S3 được thiết lập private dùng trong dự án.
  • GEMINI_API_KEY: Khóa API kết nối tới mô hình AI Google Gemini để hỗ trợ tính năng chatbot hỏi đáp.
⚠️ Cảnh báo Bảo mật:
Tuyệt đối **không push tệp .env** chứa mã bảo mật thật lên GitHub. Chỉ dùng tệp .env.example để làm mẫu và khai báo các khóa thực tế trong bảng điều khiển Environment Variables của Render.
🚀

Quy Trình Triển Khai Từng Bước

1
Đẩy mã nguồn lên GitHub
Tạo một repository riêng tư (Private Repository) trên GitHub và push toàn bộ mã nguồn của dự án lên nhánh main.
2
Đăng nhập Render & Liên kết GitHub
Truy cập dashboard.render.com, tạo tài khoản và thực hiện liên kết với tài khoản GitHub của bạn.
3
Tạo Web Service mới
Nhấp vào nút New + -> chọn Web Service. Chọn repository CloudStudy từ danh sách liên kết.
4
Cấu hình các tham số
  • Name: cloud-study
  • Runtime: Node
  • Build Command: npm install
  • Start Command: node app.js
5
Khai báo Biến Môi Trường
Nhấp vào thẻ Environment, nhấn Add Environment Variable và thêm đầy đủ các biến đã liệt kê ở phần trên, sau đó nhấn **Save changes**.
6
Deploy và Kiểm tra Logs
Render sẽ tự động kéo code về cài đặt và chạy. Theo dõi bảng điều khiển **Logs** ở bên phải màn hình để xác nhận máy chủ đã khởi tạo thành công ở địa chỉ Web URL được Render cấp.
⏱️

Khắc Phục Lỗi Ngủ Đông (Cold Start) của Render Free Tier

Trên gói dịch vụ Free của Render, nếu Web Service không nhận bất kỳ lượt truy cập nào trong vòng 15 phút, server sẽ tự động chuyển sang chế độ ngủ đông (spin down) để tiết kiệm tài nguyên. Lần truy cập tiếp theo sẽ phải chờ khoảng 30 - 50 giây để server khởi động lại (Cold Start).

Giải pháp tối ưu: Sử dụng dịch vụ ping ngoài miễn phí (ví dụ như UptimeRobot hoặc Cron-job.org) để tự động gửi HTTP Request ping tới URL website của bạn mỗi 10 - 12 phút. Việc này giúp giữ cho máy chủ luôn thức và phản hồi tức thì khi giảng viên mở kiểm tra.

💡 Các bước cài đặt UptimeRobot giữ server hoạt động:
  1. Đăng ký tài khoản miễn phí tại uptimerobot.com.
  2. Nhấp chọn Add New Monitor.
  3. Chọn Monitor Type là HTTP(s).
  4. Đặt Friendly Name là CloudStudy Live Ping.
  5. Điền địa chỉ URL website chạy thực tế được Render cấp (ví dụ: https://cloudstudy.onrender.com/login).
  6. Thiết lập tần suất ping (Monitoring Interval) là Every 10 minutes.
  7. Nhấn Create Monitor để hoàn tất.
🛠️

Lỗi Thường Gặp & Cách Khắc Phục

  • Lỗi: MONGODB_URI connection timeout / database error
    Nguyên nhân: Do MongoDB Atlas chặn dải IP lạ. Địa chỉ IP của Render server thay đổi liên tục.
    Cách xử lý: Truy cập vào trang quản trị MongoDB Atlas -> Network Access -> Cài đặt dải IP được phép truy cập là 0.0.0.0/0 (Cho phép truy cập từ mọi nơi).
  • Lỗi: AWS S3 InvalidAccessKeyId / SignatureDoesNotMatch
    Nguyên nhân: Sai thông tin Access Key hoặc Secret Key khi sao chép vào môi trường Render.
    Cách xử lý: Kiểm tra và khai báo lại biến trong phần cấu hình Environment Variables. Chú ý không sao chép dư khoảng trắng ở hai đầu chuỗi.
  • Lỗi: Node Version Mismatch / Build thất bại
    Nguyên nhân: Render sử dụng phiên bản Node.js cũ hơn mã nguồn yêu cầu.
    Cách xử lý: Thêm một biến môi trường tên là NODE_VERSION trong phần Environment của Render với giá trị phiên bản Node mong muốn (ví dụ: 20.x hoặc 22.x).
