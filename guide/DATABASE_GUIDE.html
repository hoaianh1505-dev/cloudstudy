Mô Hình Cơ Sở Dữ Liệu MongoDB

Tài liệu mô tả chi tiết lược đồ dữ liệu (Database Schema), kiểu dữ liệu và mối quan hệ giữa các bảng dùng trong dự án CloudStudy.

🍃

Giới thiệu MongoDB & Mongoose

Dự án sử dụng cơ sở dữ liệu phi quan hệ MongoDB Atlas (Cloud NoSQL) làm nơi lưu trữ dữ liệu. Giao tiếp giữa NodeJS Express và MongoDB được quản lý thông qua thư viện Mongoose ODM. Lược đồ dữ liệu được phân chia thành 3 Collection chính: users (người dùng), folders (thư mục học tập) và documents (tệp tin lưu trữ).
👤

Lược đồ Người dùng (User Schema)

Lưu trữ thông tin tài khoản đăng nhập của sinh viên và giảng viên. Mật khẩu được mã hóa an toàn bằng thuật toán bcryptjs trước khi lưu.

Trường (Field) Kiểu dữ liệu (Type) Mặc định / Ràng buộc Mô tả
_id ObjectId Tự động sinh (Primary Key) ID định danh duy nhất của người dùng.
username String Required, Trim Tên hiển thị của sinh viên.
email String Required, Unique, Lowercase Thư điện tử dùng để đăng nhập. Không trùng lặp.
password String Required Mật khẩu đã được mã hóa băm muối (bcrypt hash).
createdAt Date default: Date.now Thời gian khởi tạo tài khoản.
Mongoose Model (models/User.js) 
import mongoose from 'mongoose';

const userSchema = new mongoose.Schema({
  username: {
    type: String,
    required: true,
    trim: true
  },
  email: {
    type: String,
    required: true,
    unique: true,
    lowercase: true,
    trim: true
  },
  password: {
    type: String,
    required: true
  },
  createdAt: {
    type: Date,
    default: Date.now
  }
});

const User = mongoose.model('User', userSchema);
export default User;
📂

Lược đồ Thư mục (Folder Schema)

Mô hình hóa cấu trúc cây thư mục phân cấp lồng nhau (Toán cao cấp, Tin đại cương, Lập trình hướng đối tượng...). Mối quan hệ phân cấp cha-con được biểu diễn thông qua thuộc tính tự tham chiếu parentFolder.

Trường (Field) Kiểu dữ liệu (Type) Mặc định / Ràng buộc Mô tả
_id ObjectId Tự động sinh (Primary Key) ID định danh duy nhất của thư mục.
name String Required, Trim Tên thư mục (ví dụ: "Bài tập lớn").
parentFolder ObjectId Ref: 'Folder' Trỏ tới ID của thư mục cha. Nếu bằng null, thư mục nằm ở cấp gốc (Root).
owner ObjectId Ref: 'User', Required Trỏ tới ID người dùng sở hữu thư mục này.
createdAt Date default: Date.now Thời gian tạo thư mục.
Mongoose Model (models/Folder.js) 
import mongoose from 'mongoose';

const folderSchema = new mongoose.Schema({
  name: {
    type: String,
    required: true,
    trim: true
  },
  parentFolder: {
    type: mongoose.Schema.Types.ObjectId,
    ref: 'Folder',
    default: null
  },
  owner: {
    type: mongoose.Schema.Types.ObjectId,
    ref: 'User',
    required: true
  },
  createdAt: {
    type: Date,
    default: Date.now
  }
});

const Folder = mongoose.model('Folder', folderSchema);
export default Folder;
📄

Lược đồ Tài liệu (Document Schema)

Lưu trữ siêu dữ liệu (metadata) của tệp tin tải lên. Trường s3Key đóng vai trò cực kỳ quan trọng, là khóa định danh duy nhất của tệp được lưu vật lý trên AWS S3 Bucket.

Trường (Field) Kiểu dữ liệu (Type) Mặc định / Ràng buộc Mô tả
_id ObjectId Tự động sinh (Primary Key) ID định danh duy nhất của tài liệu.
fileName String Required, Trim Tên file gốc do người dùng tải lên (ví dụ: "de_thi_oops.pdf").
fileType String Required MIME Type của file (ví dụ: application/pdf, image/png).
fileSize Number Required Kích thước file tính bằng đơn vị Bytes.
s3Key String Required Đường dẫn khóa lưu trên S3 (ví dụ: uploads/1718816-de_thi.pdf).
s3Url String Required URL liên kết trực tiếp của S3 (để tham khảo hoặc ký tên).
folderId ObjectId Ref: 'Folder' ID thư mục cha chứa file này. Nếu bằng null, file nằm ở ngoài cùng.
owner ObjectId Ref: 'User', Required ID người dùng tải file lên hệ thống.
uploadedAt Date default: Date.now Ngày giờ tải file lên.
Mongoose Model (models/Document.js) 
import mongoose from 'mongoose';

const documentSchema = new mongoose.Schema({
  fileName: {
    type: String,
    required: true,
    trim: true
  },
  fileType: {
    type: String,
    required: true
  },
  fileSize: {
    type: Number,
    required: true
  },
  s3Key: {
    type: String,
    required: true
  },
  s3Url: {
    type: String,
    required: true
  },
  folderId: {
    type: mongoose.Schema.Types.ObjectId,
    ref: 'Folder',
    default: null
  },
  owner: {
    type: mongoose.Schema.Types.ObjectId,
    ref: 'User',
    required: true
  },
  uploadedAt: {
    type: Date,
    default: Date.now
  }
});

const Document = mongoose.model('Document', documentSchema);
export default Document;

Mối Quan Hệ Mạng Lưới (Mongoose Relationships)

Mô tả chi tiết cách ràng buộc và liên kết kho dữ liệu:

  • Một người dùng (User) sở hữu **Nhiều Thư mục (Folder)** và **Nhiều Tài liệu (Document)** (Mối quan hệ 1-N). Khai báo trong cơ sở dữ liệu bằng cách sử dụng tham chiếu ngược ref: 'User' từ các bảng con.
  • Cây thư mục phân cấp (Folder Tree): Thuộc tính parentFolder có quan hệ tự liên kết (Self-reference) 1-N về chính Model Folder. Nhờ cấu trúc này, hệ thống dễ dàng đệ quy tìm kiếm thư mục cha và danh sách các thư mục con cấp dưới ở bất kỳ độ sâu nào.
  • Quan hệ Thư mục - Tài liệu (Folder - Document): Một thư mục có thể chứa nhiều tài liệu. Trong lược đồ Document, thuộc tính folderId tham chiếu đến Model Folder, hỗ trợ truy xuất nhanh chóng các file nằm trong thư mục học tập tương ứng.
