Triết lý thiết kế mã nguồn

Dự án hướng tới sự rõ ràng, dễ bảo trì và dễ mở rộng. Toàn bộ mã nguồn sử dụng cơ chế module hóa ES Modules (sử dụng import/export của Node.js hiện đại), chia tách rạch ròi giữa các lớp dữ liệu (Models), logic định tuyến (Routes), logic trung gian (Middlewares), xử lý giao dịch dữ liệu (Services) và điều hướng hiển thị (Controllers).
📁

Sơ Đồ Cấu Trúc Thư Mục Dự Án

Dưới đây là sơ đồ hình cây chi tiết của mã nguồn ứng dụng:

CloudStudy/ // Thư mục gốc dự án
  ├── 📁 config/ // Chứa file cấu hình kết nối Database
  ├── 📁 controllers/ // Lớp điều phối dữ liệu và điều hướng trang (HTTP)
  ├── 📁 middlewares/ // Chứa bộ lọc logic (Auth check, file upload filter, folder tree parser)
  ├── 📁 models/ // Lược đồ định nghĩa cấu trúc dữ liệu Mongoose Schemas
  ├── 📁 public/ // Tài nguyên tĩnh cho trình duyệt (CSS, Front-end JS, Images)
  │   ├── 📁 css/ // Chứa các file giao diện CSS chính
  │   └── 📁 js/ // Scripts xử lý giao tiếp AJAX client (chat AI, rename, delete)
  ├── 📁 routes/ // Định nghĩa đường dẫn và ánh xạ endpoint tới Controllers
  ├── 📁 services/ // Chứa toàn bộ LOGIC nghiệp vụ chính (AWS S3, Gemini, Database transactions)
  ├── 📁 utils/ // Các hàm bổ trợ dùng chung (ví dụ: chuyển dung lượng file sang KB/MB)
  ├── 📁 views/ // Chứa giao diện hiển thị mẫu EJS (Embedded JavaScript Templates)
  │   └── 📁 partials/ // Chứa các khối HTML tái sử dụng (Header, Footer, Sidebar, Folder Tree)
  ├── 📄 app.js // File khởi tạo, liên kết cấu hình và chạy máy chủ Express
  ├── 📄 render.yaml // File Blueprint deploy tự động lên mây Render
  └── 📄 package.json // Liệt kê các thư viện phụ thuộc và scripts khởi chạy

Mô hình Service-Controller (Tách biệt Nghiệp vụ)

Nhằm đảm bảo dự án tối giản và sạch sẽ nhất, mã nguồn áp dụng triệt để kiến trúc Service-Controller Pattern:

  • Controllers (Bộ điều hướng): Chỉ làm nhiệm vụ nhận yêu cầu (Requests) từ khách hàng, trích xuất dữ liệu tham số (params, query, body), gọi tới lớp Service tương ứng, nhận về kết quả và trả về phản hồi (Response) dưới dạng hiển thị (render trang EJS) hoặc dữ liệu JSON. Tuyệt đối không chứa câu truy vấn cơ sở dữ liệu hoặc logic gọi API ngoài.
  • Services (Xử lý nghiệp vụ): Nơi thực hiện công việc nặng nhọc và tính toán. Lớp Service đảm nhiệm:
    • Truy vấn trực tiếp MongoDB (tạo, cập nhật, xóa, tìm kiếm dữ liệu).
    • Xử lý logic truyền nhận dữ liệu với AWS S3 SDK (PutObject, GetObject, DeleteObject).
    • Logic kết nối và truy vết ngữ cảnh câu hỏi với Google Gemini API.
💡 Lợi ích cho giảng viên:
Cơ chế tách biệt này giúp mã nguồn có cấu trúc rõ ràng, dễ dàng viết các bài kiểm tra tự động (Unit Tests) cho từng dịch vụ riêng biệt mà không cần giả lập (mock) các request hay response phức tạp của Express.

Giao diện với EJS & Partials

Dự án sử dụng cơ chế render giao diện tại Server (Server-Side Rendering) thông qua View Engine EJS. Các thành phần giao diện lặp lại được chia nhỏ vào thư mục views/partials/:

  • header.ejs: Khai báo cấu trúc HTML chuẩn, nạp các thư viện CSS (TailwindCSS v3 CDN, Font Awesome, Custom Main style) và cấu hình thẻ meta SEO.
  • footer.ejs: Bản quyền chân trang và liên kết các file Javascript chung của dự án.
  • sidebar.ejs: Thanh điều hướng bên trái đồng bộ trên tất cả các trang, hiển thị thông tin người dùng đang đăng nhập và nút Đăng xuất.
  • folderTree.ejs: Thành phần chứa cây thư mục đệ quy động, cho phép hiển thị cấu trúc thư mục lồng nhau trực quan ở bất kỳ cấp độ nào.
